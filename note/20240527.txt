HTTP 요청 메서드
- 클라이언트가 서버에 데이터를 전송하여 응답을 얻고자 할 때 사용하는 방식

GET : 서버의 자원을 가져오고자 할 때 사용
   ㄴ 데이터를 서버로 보내야 한다면 쿼리스트링을 사용
POST : 서버에 자원을 새로 등록하고자할 때 사용
PATCH : 자원의 일부분만 수정하는데 사용
PUT : 리소스를 수정하는 메서드 없으면 덮어쓰기 됨
DELETE : 리소스를 제거하는 코드

서블릿(Servlet)
- 자바를 기반으로 하는 웹 어플리케이션 프로그래밍 기술
- Servlet 기술을 발전시킨 것이 스프링이다
- 스프링은 실제로 Servlet을 이용하여 동작
- 자바 클래스 형태로 웹 어플리케이션을 작성하기 때문에 확장자가 .java이다
- Java를 기반으로 HTML을 첨가해서 사용
- 서블릿은 Java 코드를 작성하고나서 실행하면 class 파일을 만든다

- 서블릿의 단점은 JAVA 코드가 한 줄만 변경되어도 다시 처음부터 실행해야 한다
- 서블릿 기술에서 웹 어플리케이션을 구현하기 위해 작성해야하는 코드는 Servlet class이다

- 서블릿은 서블릿 클래스로부터 만들어진다
- 웹 컨테이너(톰캣)는 서블릿 클래스를 가지고 서블릿 객체를 만든다음 객체를 초기화하여 웹 서비스를 이용할 수 있는 상태로 만든다

JSP
- JavaServerPage의 약자로 HTML 코드에서 Java 코드를 사용하여 동적인 웹페이지를 생성하는 라이브러리
- JSP 페이지 하나가 하나의 클래스나 다름없기 때문에 모든 Java 라이브러리를 다 쓸 수 있다.

Servlet 복잡한 로직 구현에 적합함
JSP HTML 코드 중심 -> 화면 작성, 작업, 수정에 용이

jsp에 자바 코드 넣는법
스크립트릿 <% ... %> : 자바 로직을 작성할 때 사용
선언문 <%! ... %> : 자바 변수나 메서드 정의
표현문 <%= ... %> : 문자열 형태로 출력할 때 사용

page 디렉티브 태그
JSP 페이지에 대한 정보를 설정하는 태그
JSP 페이지가 생성할 콘텐츠 유형의 문서, 사용할 자바 클래스, 오류 페이지 설정, 세션 사용 여부, 출력 버퍼의 존재 유무와 같이 JSP 컨테이너가 JSP 페이지를 실행하는 데 필요한 정보를 설정

include 디랙티브 태그
- JSP 페이지의 특정 영역에 외부 파일의 내용을 포함하는 태그
포함할 수 있는 외부 파일은 HTML, JSP, 텍스트 파일 등이 있다.
include 디랙티브 태그는 jsp페이지의 어디에서든 선언할 수 잇다.
include 디랙티브 태그는 서블릿 프로그램으로 번역될 때 jsp 페이지와 설정된 다른 외부 파일의 내용이 병합되어 번역됨

jsp 내장객체
내장객체란 스크립트릿안에서만 사용할 수 있는 객체로 개발자가 별도로 생성하지 않아도 JSP에서 바로 생성할 수 있는 객체
컨테이너가 jsp를 Servlet으로 변활할 때 자동으로 객체가 생성
JSP 페이지에 별도로 import문 없이 자유롭게 사용할 수 있다.

request 요청 정보가 담겨있는 객체
response 응답할 때 사용하는 객체
session HttpSession을 구현한 객체, 세션 정보 저장에 사용
out 응답에 포함된 내용을 출력할 때 사용

response
사용자의 요청을 처리한 결과를 서버에서 웹 브라우저로 전달하는 정보를 저장
서버는 응답 헤더와 요청 처리 결과 데이터를 웹 브라우저로 보낸다

페이지 이동 관련 메서드
response.sendRedirect(String url)
처음 요청받은 현재 JSP 페이지로부터 이동할 URL을 웹브라우저로 반환
클라이언트가 새로 페이지를 요청한 것과 같은 방식으로 페이지가 이동

VO(Value Object) : 한 개 또는 그 이상의 속성들을 묶은 객체를 의미, DB를 연동하고 사람의 정보를  db에서 가져왔다고 가정했을 때 jsp의 body 부분에서 출력을 하게 된다.
